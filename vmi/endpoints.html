AGCO South America APIs

Vendor Management Inventory Integration

Recursos e operações

Essa sessão contém os campos e regras das interfaces disponibilizadas para capturar informações de peças das concessionárias.

1. Interface Inventory Data

Exemplo de JSON para a interface Inventory Data

    {
      "data": {
        "dealerLegalNumber": "99.999.999/0001-99",
        "extractionDateTime": "2016-11-23T10:45:00+03:00",
        "part": {
          "partNumber": "1444437P",
          "availableQuantity": 20,
          "onOrderQuantity": 1.5,
          "reservedQuantityWorkOrders": 1.5,
          "reservedQuantityPartTickets": 1.5,
          "openCustomerOrders": 1.5,
          "quantityReturnedByDealer": 10,
          "quantityReturnedByCustomerCounter": 1.5,
          "quantityReturnedByCustomerShop": 1.5,
          "quantityReturnedTotal": 1.5
        }
      }
    }

Especificação dos atributos

ID Campo Pai Tipo Descrição Obrigatório Regra Exemplo
A01 data Raiz TAG raiz da mensagem
A02 dealerLegalNumber varchar(50) CNPJ da concessionária "99.999.999/0001-99"
A03 extractionDateTime timestamp Data da extração dos dados da mensagem P001 "2016-11-23T10:45:00+03:00"
A04 parts A01 Grupo que contém as informações de peças
A05 partsNumber A04 varchar(50) Código do item vendido pela concessionária.Código original da concessionária "1444437P"
A06 availableQuantity A04 number(10,2) Quantidade de itens disponíveis no estoque para venda 20
- Excluir desse número peças que estão reservadas para ordens de serviços e que ainda não foram contadas como venda.
- Excluir desse número peças que estão reservadas para atender pedidos de venda.
- Excluir desse número peças pendentes de transferências
A07 onOrderQuantity A04 number(10,2) Quantidade de itens com pedidos de compras em aberto ou transferências entre filiais pendentes e que não foram baixados no estoque. 15
1.1 Pedidos
- Contar todas as peças em pedidos de compra, independente do tipo de pedido (emergência, estoque, etc), tirando os de compras realizados especificamente para um cliente.
1.2 Transferências
- Contar todos os pedidos pendentes de transferências internas, tirando os itens que já estão alocados para um cliente específico, assim como itens que estão em pedidos de balcão ou oficina
A08 reservedQuantityWorkOrders A04 number(10,2) Itens reservados para ordens de serviços abertas na oficina e que ainda não foram contados como venda. 1.5
A09 reservedQuantityPartTickets A04 number(10,2) Itens reservados para pedidos de balcão e que ainda não foram contados como venda. 1.5
A10 openCustomerOrders A04 number(10,2) Soma de itens em pedidos em aberto para cliente final. 1.5
A11 quantityReturnedByDealer A04 number(10,2) Itens retornados da concessionária para a AGCO. 10
A12 quantityReturnedByCustomerCounter A04 number(10,2) Itens retornados por pedidos de balcão para a concessionária. 1.5
A13 quantityReturnedByCustomerShop A04 number(10,2) Itens retornados por pedidos de oficina para a concessionária. 1.5
A14 quantityReturnedTotal A04 number(10,2) Soma de itens retornados por pedidos de oficina e balcão para a concessionária. 1.5

Especificação dos atributos

ID Campo Pai Tipo Descrição Obrigatório Regra Exemplo
B01 data Raiz TAG raiz da mensagem
B02 dealerLegalNumber B01 varchar(50) CNPJ da concessionária "99.999.999/0001-99"
B03 extractionDateTime B01 timestamp Data da extração dos dados da mensagem P001 "2016-11-23T10:45:00+03:00"
B04 part B01 Grupo que contém as informações de peças
B05 partsNumber B04 varchar(50) Código do item. Código original da concessionária "1444437P"
B06 agcoPartNumber B04 varchar(50) Código do item AGCO. Se o item não for um item original AGCO a concessionária deve ter um código AGCO que corresponda ao item em questão "1480626M1"
B07 originalPart B04 boolean Informação que define se o item é um item original AGCO ou não V001 true
B08 description B04 varchar(60) Descrição da peça da concessionária "REPARO PLATO C/ ALAVANCA 8 VELOCIDADES"
B09 dealerNote B04 varchar(60) Observações da concessionária para o item "Parafuso 12X70 MM Locação Obsoletos"
B10 stockable B04 boolean É um item estocável ou não V002 true
B11 lastSaleDate B04 date Última data de venda da peça (data de nota fiscal) "2016-11-23"
B12 mainBinLocation B04 varchar(60) Local principal de armazenamento da peça na concessionária. Se não existe a informação para o item enviar apenas os delimitador R001 "E 03 03"
B13 alternateBinLocation B04 varchar(60) Local secundário de armazenamento da peça na concessionária. "E 03 03"
B14 minimumThreshold B04 number(10,2) Estoque mínimo para o item na concessionária R001 1,5
B15 maximumThreshold B04 number(10,2) Estoque máximo para o item na concessionária R001 1,5
B16 preferredSupplierLegalNumber B04 varchar(50) CNPJ do fornecedor do item "00.000.000/0001-00"
B17 preferredSupplierName B04 varchar(100) Nome fantasia do fornecedor preferencial para o item "NOME FANTASIA DO FORNECEDOR"
B18 netPrice B04 number(14,2) Preço de lista do item na concessionária 200,00
B19 averageCost B04 number(14,2) Custo médio do item na concessionária 200,00
B20 unitOfMeasure B04 varchar(3) Unidade de medida de armazenamento do item "M"
B21 dealerPartsPerPackage B04 number(14,2) 1,5
B22 currencyCode B04 varchar(3) Código da moeda P002 "BRL"
B23 lastPurchasePrice B04 number(14,2) Último preço de venda pago 200.00
B24 openPurchaseOrders B04 number Quantidade de itens em pedidos de compras abertos 10
B25 vendorPackageQuantity B04 number Informação apenas para itens não AGCO:Embalagem mínima que o item é solicitado no fornecedor 10
B26 segmentationCode01 B04 varchar(30) Campo para informações de segmentação da concessionária "FLT"
B27 segmentationCode02 B04 varchar(30) Campo para informações de segmentação da concessionária
B28 segmentationCode03 B04 varchar(30) Campo para informações de segmentação da concessionária
B29 segmentationCode04 B04 varchar(30) Campo para informações de segmentação da concessionária
B30 firstAvailableDate B04 date Data da primeira movimentação do item na filial. Opção 1: enviar a data que a peça foi recebida pela primeira vez no estoque. Opção 2: se não tiver a informação da opção 1 enviar a data que a peça foi solicita pela primeira vez pela oficina. Opção 3: se não tiver a informação das opções 1 e 2 enviar a data que a peça foi adicionada pela primeira vez no estoque. Opção 4: se não tiver a informação das opções 1, 2 e 3 enviar a data 1900-01-01 "2016-11-23"
B31 firstPurchase B04 boolean Esse campo identifica se essa peça nunca esteve em estoque porque é um registro de primeira compra V003 false

D25 quantityAvailable D13 number(14,2) Quantidade de itens disponíveis na data que o pedido foi solicitado 20

D27 serialNumber D13 varchar(30) Série comercial da maquina que esse item será utilizado
